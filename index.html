      대장길드 참여자 체크 시스템 상세 설명서
      1. 메인페이지 주요 기능
      1.1 점수 현황
      • 누적 점수
        • 전체 기간 동안의 참여자별 총점과 참여율을 확인할 수 있습니다.
        • 참여율은 전체 보스 중 참여한 보스의 비율로 계산됩니다.
        • 예시: 총 100개의 보스 중 80개에 참여했다면 참여율은 80%입니다.
      • 주간 점수
        • 최근 7일간의 참여자별 점수와 참여율을 확인할 수 있습니다.
        • 주간 점수는 매일 자정에 자동으로 초기화됩니다.
        • 주간 참여율은 최근 7일간의 보스 중 참여한 보스의 비율로 계산됩니다.
      • 기간별 점수
        • 원하는 기간을 직접 설정하여 해당 기간의 점수와 참여율을 확인할 수 있습니다.
        • 시작일과 종료일을 선택하여 특정 기간의 데이터만 조회할 수 있습니다.
        • 기간별 데이터는 CSV로 다운로드하여 저장할 수 있습니다.
      • 정렬 기능
        • 점수순 또는 참여율순으로 정렬하여 확인할 수 있습니다.
        • 오름차순/내림차순 정렬이 가능합니다.
        • 정렬된 데이터는 CSV로 다운로드할 수 있습니다.
      1.2 닉네임 설정
      • 닉네임 저장
        • 메인페이지 상단의 닉네임 입력란에 자신의 닉네임을 입력하고 저장합니다.
        • 저장된 닉네임은 랜덤 뽑기, 당번 신청 등에서 자동으로 사용됩니다.
        • 닉네임은 브라우저에 저장되므로 다음 접속 시에도 유지됩니다.
      • 닉네임 변경
        • 저장된 닉네임을 변경하려면 새로운 닉네임을 입력하고 저장합니다.
        • 닉네임 변경 시 이전 닉네임의 데이터는 유지됩니다.
        • 관리자는 상세 수정 메뉴에서 닉네임을 일괄 변경할 수 있습니다.
      1.3 오픈된 보스
      • 보스 목록
        • 현재 진행 중인 보스 목록과 각 보스별 참여 현황을 확인할 수 있습니다.
        • 보스 오픈 시간으로부터 6시간 동안 참여 선택이 가능합니다.
        • 참여 가능 시간이 지난 보스는 자동으로 목록에서 제외됩니다.
      • 참여 현황
        • 각 보스별 참여자 수와 참여율을 확인할 수 있습니다.
        • 참여자 목록을 클릭하면 상세 참여자 정보를 볼 수 있습니다.
        • 참여 현황은 실시간으로 업데이트됩니다.
      • CSV 다운로드
        • 현재 보스 목록을 엑셀 파일로 저장할 수 있습니다.
        • 파일명에 날짜가 자동으로 포함됩니다.
        • CSV 파일에는 보스 ID, 오픈 시간, 참여자 수, 참여율이 포함됩니다.
      2. 메뉴별 상세 기능
      2.1 랜덤 뽑기 & 분배 내역
      • 기본 아이디 목록
        • 자주 사용하는 아이디들을 미리 등록해두고 관리할 수 있습니다.
        • 전체추가 버튼으로 한 번에 모든 아이디를 뽑기 대상에 추가할 수 있습니다.
        • 기본 아이디 목록은 관리자만 수정할 수 있습니다.
      • 뽑기 대상 관리
        • 기본 아이디 목록에서 개별 선택하여 추가할 수 있습니다.
        • 단체 추가 기능으로 여러 닉네임을 한 번에 추가할 수 있습니다.
        • 예시: "총괄 수달 젤리" 입력 시 → "총괄대장", "수달이", "곰돌이젤리" 등이 자동으로 매칭됩니다.
        • 뽑기 대상 목록은 페이지 새로고침 시 초기화됩니다.
      • 추첨 기능
        • 뽑을 인원 수를 입력하고 추첨하기 버튼을 클릭합니다.
        • 추첨 결과는 화면에 즉시 표시되며, 분배 내역에 자동으로 기록됩니다.
        • 추첨 결과는 CSV로 다운로드할 수 있습니다.
      • 분배 내역 관리
        • 날짜, 닉네임, 아이템 정보를 확인할 수 있습니다.
        • CSV 다운로드로 전체 분배 내역을 엑셀 파일로 저장할 수 있습니다.
        • 분배 내역은 Firebase에 저장되므로 영구적으로 보관됩니다.
      2.2 당번 신청
      • 날짜별 당번 관리
        • 원하는 날짜를 선택하여 당번을 신청할 수 있습니다.
        • 일일 최대 인원 수를 설정하여 관리할 수 있습니다.
        • 당번 신청은 선착순으로 처리됩니다.
      • 전체 시간 설정
        • 모든 날짜에 공통으로 적용될 시간을 한 번에 설정할 수 있습니다.
        • 개별 날짜의 시간도 수정 가능합니다.
        • 시간 설정은 24시간 형식으로 입력합니다.
      • 관리자 기능
        • 당번 초기화: 특정 날짜의 당번 정보를 초기화할 수 있습니다.
        • 전체 초기화: 모든 당번 정보를 초기화할 수 있습니다.
        • 당번 정보는 Firebase에 저장되므로 영구적으로 보관됩니다.
      2.3 길드 자금 현황
      • 자금 내역 관리
        • 내용, 금액, 증감 구분, 비고를 입력하여 자금 내역을 기록합니다.
        • 잔액이 자동으로 계산되어 표시됩니다.
        • 자금 내역은 Firebase에 저장되므로 영구적으로 보관됩니다.
      • CSV 다운로드
        • 전체 자금 내역을 엑셀 파일로 저장할 수 있습니다.
        • 날짜별로 파일명이 자동 생성됩니다.
        • CSV 파일에는 날짜, 내용, 금액, 증감 구분, 비고, 잔액이 포함됩니다.
      3. 관리자 전용 기능
      3.1 상세 수정
      • 누락 데이터 추가
        • 보스 ID, 시간, 닉네임, 점수를 입력하여 누락된 데이터를 추가할 수 있습니다.
        • 추가된 데이터는 즉시 점수 현황에 반영됩니다.
        • 누락 데이터 추가는 관리자만 가능합니다.
      • 참여자 관리
        • 보스별 참여자 목록 확인 및 수정
        • 점수 수정, 닉네임 변경, 참여자 추가 기능
        • 단체 추가 기능으로 여러 참여자를 한 번에 추가
        • 참여자 관리 기능은 관리자만 사용 가능합니다.
      • 보스 ID 관리
        • 보스 ID 변경 기능
        • 기존 데이터는 새로운 ID로 이전됩니다.
        • 보스 ID 관리는 관리자만 가능합니다.
      4. 주의사항 및 팁
      • 데이터 저장
        • 모든 데이터는 Firebase에 실시간으로 저장됩니다.
        • 인터넷 연결이 불안정할 경우 데이터 저장이 지연될 수 있습니다.
        • 중요한 데이터는 주기적으로 CSV로 다운로드하여 백업하세요.
      • 닉네임 매칭
        • 두 글자 이상의 부분 문자열로 실제 아이디를 찾을 수 있습니다.
        • 띄어쓰기로 구분하여 여러 닉네임을 한 번에 입력할 수 있습니다.
        • 예시: "총괄 수달 젤리" → "총괄대장", "수달이", "곰돌이젤리" 매칭
      • 데이터 백업
        • 중요한 데이터는 주기적으로 CSV로 다운로드하여 백업하세요.
        • 뽑기/참여자 목록은 페이지 새로고침 시 초기화될 수 있습니다.
        • 분배 내역, 당번 정보, 자금 내역은 Firebase에 저장되므로 영구적으로 보관됩니다.
      5. 문의 및 피드백
      • 시스템 사용 중 오류가 발생하거나 개선이 필요한 부분이 있다면 관리자에게 직접 전달해 주세요.
      • 새로운 기능 제안이나 사용자 경험 개선 의견도 환영합니다.
      • 문의사항은 관리자에게 직접 전달해 주세요.
